Research by Institute of Computing/Federal University of Alagoas (IC/UFAL)

Guidelines: Good pratices in C programs with ifdefs through bugs in real code.
Projects: GCC, Gnome, Hurd, Gimp, Linux, Libreoffice, Firefox, Wireshark
Bugs/Guidelines description by colors:

  • Bug #1: x = 3; <-> x = 4;
  • Bug #2: int x; <-> char x;
  • Bug #3: #ifdef A int x; #endif <-> int x; #ifdef B #endif
  • Bug #4: function(void) <-> function(int)
  • Bug #5: function() <-> function(3)
  • Bug #6: #ifdef A int x; #endif <-> #ifdef B function(x) #endif
  • Bug #7: #ifdef A int x; #endif <-> int x
  • Bug #8: function(x) <-> function(x, y)
  • Bug #9: if { #ifdef A } else { #ifdef A }
  • Bug #10: #ifdef A #include x #endif <-> #include x #ifdef A x #endif
